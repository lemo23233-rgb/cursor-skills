- **产品一句话**：为普通用户提供「输入故事主题 → 自动生成故事大纲与分镜文案 → 一键生成多张连贯插画」的微信小程序，风格一致，适合做简易故事书/连环画。

## 五、技术方案

### 5.1 总体框架（前后端）

- **前端**：微信小程序（miniprogram），技术栈为小程序原生 + wx.cloud 调用云函数；页面包括首页、配置页、创作页、作品详情页。
- **后端**：腾讯云 CloudBase 云函数（Node.js），负责接收小程序请求、校验参数、使用用户提供的 API Key 转发至火山方舟，不持久化 API Key；**作品数据**由小程序或云函数写入 CloudBase 文档型数据库，列表与最近作品从数据库读取。
- **数据流**：小程序将 API Key 与业务参数通过云函数调用传入 → 云函数使用该 Key 以 Bearer 方式请求火山方舟文本生成 / 图片生成 API → 将结果返回小程序；生成完成后作品写入腾讯云数据库，首页/作品列表从数据库读写。

```
┌─────────────────┐     wx.cloud.callFunction      ┌──────────────────┐     HTTPS (Bearer API Key)     ┌─────────────────┐
│  微信小程序      │  (apiKey + topic / storyboard)  │  CloudBase 云函数  │  ───────────────────────────►  │  火山方舟        │
│  (前端)         │  ───────────────────────────►  │  (Node.js)       │  ◄───────────────────────────  │  文本生成/图片生成 │
│                 │  ◄───────────────────────────  │                  │      (大纲/分镜文案 或 图片)   │                 │
└────────┬────────┘      (JSON 结果)               └────────┬─────────┘                                └─────────────────┘
         │                                                 │
         │  wx.cloud.database() 读写作品                    │  云函数可写库（可选）
         └────────────────────►  ┌──────────────────┐  ◄──┘
                                 │  CloudBase 文档库   │
                                 │  (作品集合 stories) │
                                 └──────────────────┘
```

- **云函数划分建议**：
  - `arkText`（或 `generateStory`）：接收主题 + API Key，调用火山方舟文本生成，返回故事大纲与分镜文案。
  - `arkImage`（或 `generateStoryImages`）：接收分镜文案列表 + API Key，按序或成组调用火山方舟图片生成（doubao-seedream-4.5），返回每镜图片 URL 或 base64。

### 5.2 AI 模型接口

| 能力 | 火山方舟侧 | 说明 |
|------|------------|------|
| **鉴权** | 请求 Header：`Authorization: Bearer <ARK_API_KEY>` | 云函数从小程序传入的 `apiKey` 拼成 Bearer Token 调用方舟，Key 不落库。 |
| **文本生成** | 火山方舟「文本生成」API（Chat 接口） | 文档：[文本生成](https://www.volcengine.com/docs/82379/1399009)。请求体含 model_id（如豆包大模型）、messages（系统提示词 + 用户主题）。系统提示词约定输出结构：故事大纲 + 分镜文案（每镜一段，固定 4–6 镜）。响应解析为大纲与分镜列表返回前端。 |
| **图片生成** | 火山方舟「图片生成」API | 文档：[图片生成 API](https://www.volcengine.com/docs/82379/1541523)。模型使用 **doubao-seedream-4.5** 对应 endpoint_id/model_id（以控制台/模型列表为准）。支持成组/多图以保持风格一致，请求体含 prompt（分镜文案）、尺寸等；响应为图片 URL 或 base64 列表。 |

- **Base URL**：以火山方舟文档「Base URL 及鉴权」为准（如 `https://ark.cn-beijing.volces.com/api/v3` 等），云函数内配置或从文档读取。
- **错误与限流**：云函数将方舟返回的错误码与信息原样或转义后返回小程序，便于提示「Key 无效」「限流」等。

### 5.3 数据模型

- **前端本地存储**
  - `ark_api_key`：用户输入的火山方舟 API Key（仅本地，请求时带给云函数）。
  - 作品列表、最近作品卡片**不**存本地，均从腾讯云数据库读写。

- **腾讯云数据库（CloudBase 文档型数据库）**
  - **集合名**：建议 `stories`（作品集合）。
  - **作品文档字段**：`_id`（云数据库自动）、`topic`（主题）、`outline`（大纲）、`scenes`（分镜文案数组）、`imageUrls`（与分镜顺序一致的图片 URL 或可存 cloudPath）、`sceneCount`（分镜数，便于列表展示）、`createdAt`（生成时间）、`_openid`（小程序端写入时自动，用于按用户隔离）。
  - **读写方式**：小程序端通过 `wx.cloud.database()` 增删改查；生成完成后在创作页或云函数内执行「添加」写入一条作品；首页「最近作品」、作品列表页按 `createdAt` 降序查询，支持分页；作品详情页根据 `_id` 查询单条。

- **云函数「生成大纲与分镜」**
  - **入参**：`{ apiKey: string, topic: string }`
  - **出参**：`{ outline: string, scenes: string[] }`（`scenes` 为分镜文案数组，长度建议 4–6）

- **云函数「成组生图」**
  - **入参**：`{ apiKey: string, scenes: string[] }`（可选带风格/尺寸等）
  - **出参**：`{ images: string[] }`（`images` 为每镜图片 URL 或 base64，与 `scenes` 一一对应）

- **作品与列表**
  - 生成完成后：将 `topic`、`outline`、`scenes`、`imageUrls`、`sceneCount`、`createdAt` 写入 `stories` 集合；首页/列表页从该集合按时间倒序查询展示作品列表与最近作品卡片；详情页根据 `_id` 从数据库读取或由列表带入。

## 七、页面设计说明

### 7.1 首页（pages/index/index）

- **导航栏标题**：如「故事书助手」。
- **主区**：
  - 主按钮：「开始创作」→ 跳转创作页。
  - **最近作品卡片**：从腾讯云数据库按 `createdAt` 降序查询最近 1 条或若干条，展示摘要（主题、分镜数、首图等），点击进入作品详情；若有「更多」则进入作品列表页。
  - 入口「作品列表」：跳转作品列表页，列表数据从腾讯云数据库分页读取。
  - 底部或右上角：「设置」→ 配置页。
  - 若无作品：文案提示「还没有作品，去创作一个吧」。
- **说明文案**：简短一句「输入主题，AI 生成故事与连环画插画」，并注明「需在设置中配置 API Key 方可使用」「试用阶段，用量以火山引擎为准」。

### 7.2 配置页（pages/settings/index）

- **导航栏标题**：「设置」。
- **API Key 配置**：
  - 输入框：placeholder「请输入火山方舟 API Key」，类型 password，用于输入 Key。
  - 按钮：「保存」：将 Key 仅写入本地（如 wx.setStorageSync），并提示「保存成功，请勿泄露」。
  - 按钮：「清除」：删除本地 Key，并提示已清除。
- **文档入口**（重点）：
  - 文案：「如何获取 API Key？」
  - 可点击跳转或复制链接：**https://www.volcengine.com/docs/82379/1263279**（获取 API Key 并配置）。
  - 可选补充：开通服务说明链接 https://www.volcengine.com/docs/82379/1159200。
- **试用与责任声明**：注明当前为试用、用量与计费以火山引擎为准、请勿泄露 API Key。

### 7.3 创作页（pages/create/index）

- **导航栏标题**：「创作故事书」。
- **步骤区**（建议分步或同一页分块）：
  1. **步骤 1 - 主题**
     - 输入：单行或短多行「故事主题/梗概」。
     - 按钮：「生成大纲与分镜」。
     - 若未配置 API Key：拦截并提示去设置页配置，并给出文档链接。
  2. **步骤 2 - 大纲与分镜**
     - 展示文本模型返回的「故事大纲」与「分镜文案」（每镜一段），可支持简单编辑（文本框）。
     - 显示分镜条数（如共 N 镜）。
  3. **步骤 3 - 生成插画**
     - 按钮：「生成连环画插画」。
     - 说明：将按分镜顺序调用 doubao-seedream-4.5 成组生成多张图（模型能力以官方为准，如 2–10 张）。
     - 生成中：Loading + 进度或「第 x / N 镜」。
     - 生成完成：将作品写入腾讯云数据库（`stories` 集合），再跳转作品详情页或在本页底部展示结果。
- **异常与提示**：
  - Key 无效/过期：提示检查 Key 或重新获取。
  - 限流/错误：展示接口返回错误信息或通用「生成失败，请稍后重试」。

### 7.4 作品详情页（pages/story-detail/index）

- **导航栏标题**：「作品详情」。
- **内容**：
  - 按顺序展示：每镜的「分镜文案 + 对应生成图」。
  - 图片支持点击预览、长按保存到相册（符合微信规范）。
- **数据来源**：根据作品 `_id` 从腾讯云数据库读取该条作品；或由列表页/首页传入已查好的作品数据，避免重复请求。

### 7.5 作品列表页（pages/story-list/index）

- **导航栏标题**：「我的作品」或「作品列表」。
- **内容**：从腾讯云数据库 `stories` 集合按 `createdAt` 降序分页查询，每条展示为卡片（主题、分镜数、首图或占位图、创建时间）；点击卡片进入作品详情页（传入 `_id` 或作品数据）。
- **交互**：上拉加载更多（分页）；若无数据则提示「还没有作品，去创作一个吧」并引导至创作页。

## 八、功能需求详述

### 8.1 配置管理

- **保存 API Key**：用户输入 Key → 仅写入小程序本地存储 → 云函数调用时由前端将 Key 放在请求体或 Header 中传给云函数（云函数不落库 Key，或按需加密存、按用户隔离）。
- **清除 API Key**：删除本地 Key，后续生成请求需提示先配置。
- **获取 Key 文档**：在配置页提供固定链接 **https://www.volcengine.com/docs/82379/1263279**，建议可点击复制或使用 `wx.setClipboardData` 复制链接，并文案说明「请在浏览器中打开并登录火山引擎控制台获取 API Key」。

### 8.2 故事创作（文本生成）

- **输入**：故事主题/梗概（必填，长度限制可设 20–500 字）。
- **调用**：云函数内部调用火山方舟「文本生成」API，提示词需包含：根据主题生成「故事大纲」+「分镜文案」（每镜一段，条数可固定如 4–6 镜，便于成组生图）。
- **输出**：结构化或分段文本返回小程序，展示在创作页「步骤 2」并可编辑。

### 8.3 连环画生成（图片生成）

- **输入**：分镜文案列表（来自步骤 2，可为编辑后结果）。
- **模型**：**doubao-seedream-4.5**（请求体中使用火山方舟该模型对应 endpoint_id/model_id，以官方文档/控制台为准）。
- **调用方式**：参考 [Seedream 4.0-5.0 教程](https://www.volcengine.com/docs/82379/1824121) 与 [图片生成 API](https://www.volcengine.com/docs/82379/1541523)，使用**成组/多图**能力，保证风格与角色一致（若文档示例为 4.0，则参数改为 4.5 对应模型 ID）。
- **流程**：云函数接收分镜列表 + 用户 API Key，循环或按批量调用图片生成接口，返回每镜的图片 URL 或 base64，由小程序展示并支持保存。

### 8.4 结果展示与导出

- 作品详情页按顺序展示「分镜文案 + 图」。
- 单张图：点击预览、长按保存到相册。
- 若需「分享卡片」：需符合微信分享规范（标题、图片、路径），可不作为 MVP 必做。

### 8.5 作品列表与存储（腾讯云数据库）

- **写**：创作页生成完成后，将作品（topic、outline、scenes、imageUrls、sceneCount、createdAt）写入 CloudBase 文档型数据库集合 `stories`；小程序端调用 `db.collection('stories').add()` 或由云函数写入，需保证图片 URL 或云存储 path 可长期访问（若为临时 URL 可先上传至 CloudBase 云存储再存 path）。
- **读**：首页「最近作品」卡片：按 `createdAt` 降序、limit 1 或若干条查询；作品列表页：按 `createdAt` 降序分页查询，展示卡片列表；作品详情页：根据 `_id` 查询单条或由列表带入。
- **权限**：数据库安全规则建议按 `_openid` 隔离，仅允许用户读写自己的作品。

## 九、非功能需求

- **安全**：API Key 不写死在小程序代码中；仅通过用户输入保存到本地，请求时经云函数转发，云函数用该 Key 调方舟。
- **体验**：生成过程需 Loading、失败有明确提示；配置页文档链接醒目。
- **合规**：页面上注明试用说明、用量与计费以火山引擎为准、API Key 获取与使用责任在用户。
