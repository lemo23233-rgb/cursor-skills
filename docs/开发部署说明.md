# 开发与部署说明（MVP）

本项目为「微信小程序 + 微信云开发（CloudBase 云函数）」形态：

## 0. 小程序基本设置

在发布前，需在微信公众平台配置名称、简称、头像和介绍：

1. 登录 [微信公众平台](https://mp.weixin.qq.com)（小程序管理后台）
2. 进入 **设置** → **基本设置**
3. 在「基本信息」中填写：

| 配置项 | 说明 | 建议值 |
|--------|------|--------|
| **小程序名称** | 用户搜索/展示用，2–30 个字符，一年可修改 2 次 | 故事书助手 |
| **小程序简称** | 4–10 个字符，用于搜索与展示，一年可修改 2 次 | 故事书 |
| **小程序头像** | 144×144 px，支持 JPG/PNG，≤ 2MB | 需自行上传 |
| **小程序介绍** | 4–120 个字符，用于搜索与展示 | 输入主题，AI 生成故事与连环画插画，创作你的奇幻世界 |

4. **头像**：上传后需等待审核，建议使用简洁、易识别的图标（如书本、画笔等元素）
5. **名称**：若已存在同名或相似名称，需更换或调整；建议在公众平台搜索确认名称是否可用

完成以上配置后，用户在小程序列表、搜索、分享等场景中会看到对应的名称、简称、头像和介绍。

- 小程序端仅**本地保存**火山方舟 `API Key`，每次请求时把 `apiKey` 传给云函数
- 云函数只负责**转发**到火山方舟文本/图片接口，**不持久化** `apiKey`

## 1. 云函数目录

云函数位于：

- `cloudfunctions/arkText`：生成「故事大纲 + 分镜文案」
- `cloudfunctions/arkImage`：根据分镜文案生成插画

小程序端通过 `wx.cloud.callFunction` 调用的函数名分别是：`arkText`、`arkImage`（需与云端已部署函数名一致）。

## 2. 云函数配置（环境变量 + 超时）

### 2.1 环境变量（必配）

请在 CloudBase 控制台或微信开发者工具里，为云函数配置环境变量：

- **ARK_BASE_URL**（可选）：默认 `https://ark.cn-beijing.volces.com/api/v3`
- **ARK_TEXT_MODEL**（必填）：火山方舟「文本生成」所用模型/接入点标识，具体取值以[文本生成文档](https://www.volcengine.com/docs/82379/1399009?lang=zh)为准
- **ARK_IMAGE_MODEL**（必填）：火山方舟「图片生成」所用模型/接入点标识，以[图片生成 API 文档](https://www.volcengine.com/docs/82379/1541523)为准

说明：

- 若点击「生成大纲与分镜」无报错但无内容：请检查云函数环境变量 **ARK_TEXT_MODEL** 是否已配置且与火山方舟控制台/文档一致，以及小程序设置页中的 API Key 是否有效。

### 2.2 超时时间（必调，否则易报 -504003）

调用火山方舟接口通常需要 **20-60 秒**，默认 3 秒会触发 `Invoking task timed out (errCode: -504003)`。

- 在 **微信开发者工具**：云开发 → 云函数 → 选中 **arkText** / **arkImage** → 右键「云函数配置」或进入详情 → **超时时间** 改为 **60 秒**。
- 在 **腾讯云 CloudBase 控制台**：云函数 → 对应函数 → 函数配置 → 执行超时时间 设为 **60 秒**（小程序 callFunction 最长支持 60 秒）。

两个云函数都建议设为 **60 秒**。生图已改为「每张图单独调用云函数」，单次调用只生成一张图，一般可在 60 秒内完成，避免多图一次生成超时。

## 3. 云数据库（作品列表 / 最近作品）

作品数据存储在**云开发文档数据库**中，集合名为 `stories`。

- **集合**：在云开发控制台 → 数据库 → 创建集合 `stories`（若不存在，首次写入时会自动创建）。
- **文档字段**：`topic`、`outline`、`scenes`（数组）、`images`（数组）、`createdAt`、`updatedAt`；文档 `_id` 使用前端生成的 story id，便于按 id 查询详情。
- **权限**：请为 `stories` 配置安全规则，使**仅当前用户可读写自己的数据**（云开发会在写入时自动注入 `_openid`）。建议规则：
  - 读：`doc._openid == auth.openid`
  - 写：`doc._openid == auth.openid`
  - 或使用控制台「仅创建者可读写」模板。
- **首页「最近作品」**与**作品列表页**均从该集合按 `updatedAt` 倒序查询；**作品详情**按 `_id` 取单条；**创作完成**时写入/覆盖该集合。

## 4. 本地调试流程（微信开发者工具）

1. 打开项目根目录（包含 `project.config.json`）
2. 确保已开通并选择云开发环境
3. 在「云函数」面板找到 `arkText` / `arkImage`，分别上传并部署
4. 在「云开发」→「数据库」中创建集合 `stories`（若尚无），并配置安全规则为「仅创建者可读写」或自定义 `doc._openid == auth.openid`
5. 小程序端进入「设置」页，粘贴并保存火山方舟 `API Key`
6. 回到首页 → 开始创作 → 输入主题 → 生成大纲与分镜 → 生成插画 → 作品会写入云数据库，并可在「作品列表」与详情页从数据库加载

## 5. 常见问题

- **提示未配置 API Key**：去「设置」页保存 Key
- **云函数提示未配置 ARK_TEXT_MODEL / ARK_IMAGE_MODEL**：到云函数环境变量补齐
- **生成失败/限流**：可能是 Key 无效、余额/权限不足、或触发限流；可稍后重试或更换 Key
- **errCode: -504003 / Invoking task timed out**：云函数执行超时。请将 **arkText**、**arkImage** 的超时时间均设为 **60 秒**（见上文 2.2）。
- **点击「生成大纲」没有请求到 /chat/completions**：
  1. 在微信开发者工具或 CloudBase 控制台打开 **arkText 云函数日志**，点击一次「生成大纲与分镜」后查看是否出现 `[arkText] event keys:`、`[arkText] hasApiKey: true hasTopic: true TEXT_MODEL: set`、`[arkText] calling https://.../chat/completions`。若没有「calling」说明云函数未执行到发请求（多为未配置 ARK_TEXT_MODEL 或入参缺失）。
  2. 确认云函数环境变量 **ARK_TEXT_MODEL** 已配置且与火山方舟控制台一致。
  3. 确认小程序「设置」页已保存 API Key，且创作页已输入主题后再点「生成大纲与分镜」。
- **文生图阶段没有请求到 /images/generations**：
  1. 查看 **arkImage 云函数日志**，点击「生成连环画插画」后是否出现 `[arkImage] event keys:`、`[arkImage] hasApiKey: true scenesCount: N IMAGE_MODEL: set`、`[arkImage] calling .../images/generations scene 1/N`。若没有「calling」多为未配置 **ARK_IMAGE_MODEL** 或入参 scenes 为空。
  2. 确认云函数环境变量 **ARK_IMAGE_MODEL** 已配置；确认创作页已生成大纲与分镜且分镜不为空后再点「生成连环画插画」。

## 6. npm 依赖安装失败

### 现象

- 在项目根目录或 `miniprogram` 下执行 `npm install` 报错
- 在 `cloudfunctions/arkText` 或 `cloudfunctions/arkImage` 下执行 `npm install` 报错（如 `ENOTEMPTY: directory not empty, rename ...`）

### 常见原因与处理

1. **ENOTEMPTY / rename 失败**  
   npm 在写 `node_modules` 时发生目录重命名冲突，多为上次安装中断或异常留下脏目录。

   **处理：**
   - 在项目根目录执行（会依次清理并安装根目录与 arkText 云函数）：
     ```bash
     chmod +x scripts/fix-npm-install.sh && ./scripts/fix-npm-install.sh
     ```
   - 或手动执行：
     - **云函数**：`cd cloudfunctions/arkText` → `rm -rf node_modules package-lock.json` → `npm install`
     - **项目根**：在仓库根目录执行 `rm -rf node_modules package-lock.json` → `npm install`

2. **网络 / 镜像超时**  
   使用国内镜像可减少超时：
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```
   如需恢复默认：
   ```bash
   npm config set registry https://registry.npmjs.org
   ```

3. **云函数依赖不必在本地装**  
   微信云开发支持「上传并部署：云端安装依赖」：在开发者工具中右键云函数目录（如 `arkText`）→ 选择该选项，由云端在部署时安装依赖，本地可不执行该云函数目录下的 `npm install`。若本地也要跑或调试，再按上面 1 清理后执行 `npm install`。

4. **Node 版本**  
   建议使用 Node 16 或 18 LTS；过旧或过新可能导致部分包安装异常。  
   `node -v` 查看版本，必要时用 nvm 等切换版本后再执行 `npm install`。
